## Сводка критических замечаний из аудитов

Оба аудита сходятся в следующих фундаментальных проблемах:

1. **Глобальный толеранс 1e-6** — не масштабируется, не учитывает размер модели, нет иерархии
2. **Триангуляция не гарантирует водонепроницаемость** — общие рёбра граней дискретизируются независимо, слияние постфактум
3. **Нет модуля healing** — «лечение» сводится к удалению, а не восстановлению
4. **Слабая поддержка вырожденных случаев** — NaN/Inf, нулевые рёбра, полюса, апексы
5. **STEP-парсер не валидирует геометрию** — нет семантической проверки, нет извлечения толерансов из файла
6. **Нет адаптивной триангуляции** — `max_deviation` и `max_edge_length` объявлены, но не используются
7. **Нет аналитических производных NURBS** — нормали вычисляются численно, нет dS/du, dS/dv
8. **Нет аналитических PCURVE** — только полилинейная аппроксимация, нет Curve2d
9. **Отсутствие многоуровневой валидации топологии** — нет проверки манифольдности, ориентации, замкнутости
10. **Нет распараллеливания** — полностью однопоточная обработка

## Фаза 1: Фундамент — Толерансы и Водонепроницаемость (0–3 месяца)

> **Цель:** Устранить корневые причины негерметичных сеток и некорректной дискретизации.

### 1.1 Система контекстных толерансов

**Проблема:** Глобальные константы `TOLERANCE=1e-6`, `ANGULAR_TOLERANCE=1e-5`, `PARAMETRIC_TOLERANCE=1e-8` не адаптируются к масштабу модели.

**Задачи:**

- [x] 1.1.1 Создать структуру `ToleranceContext` в `draper-geometry/src/tolerance.rs` с полями: `absolute`, `relative`, `angular`, `parametric`, `model_scale`
- [x] 1.1.2 Метод `is_coincident(a, b) -> bool` с учётом `absolute + relative * model_scale`
- [x] 1.1.3 Метод `parametric_tolerance(surface, u, v) -> f64` — пересчёт 3D-толеранса в UV-пространство с учётом якобиана поверхности
- [x] 1.1.4 Метод `from_step_model(solid)` — автоматическое вычисление `model_scale` по диагонали bounding box
- [x] 1.1.5 Метод `from_bounding_box(bb: &BoundingBox)` — конструирование по известному bbox
- [x] 1.1.6 Заменить все использования `TOLERANCE` на `tol_ctx.absolute()` (или `tol_ctx.is_coincident()`) — backward-compatible константы сохранены
- [x] 1.1.7 Передавать `ToleranceContext` через всю цепочку: `StepConverter → triangulate_solid → triangulate_face → merge_coincident_vertices` — `ToleranceContext` создан, передача через EdgeDiscretizationCache
- [x] 1.1.8 Обновить `TriangulationParams` — убрать дублирование, хранить ссылку на `ToleranceContext`
- [x] 1.1.9 Добавить извлечение толерансов из STEP-сущностей: `GEOMETRIC_TOLERANCE`, `UNCERTAINTY_MEASURE_WITH_UNIT`, `SHAPE_TOLERANCE` — при наличии переопределять default
- [x] 1.1.10 Unit-тесты: микронная деталь (scale=0.001), метровая сборка (scale=1000), километровая сцена (scale=1e6)

**Критерий приёмки:** Ни одна функция в `draper-geometry` и `draper-mesh` не использует голую константу `TOLERANCE` напрямую — только через `ToleranceContext`.

**Проблема:** Каждая грань триангулируется независимо, общие рёбра дискретизируются дважды с разными точками, потом сливаются `merge_coincident_vertices` — но это не гарантирует водонепроницаемость.

**Задачи:**

- [x] 1.2.1 Создать структуру `EdgeDiscretizationCache` в новом модуле `draper-mesh/src/edge_cache.rs`
- [x] 1.2.2 Метод `discretize_edge(edge, face_id, surface, tol_ctx)`: если ребро уже в кэше → вернуть существующие точки + вычислить UV для данной поверхности; если нет → вычислить точки по кривой с учётом кривизны (адаптивно), сохранить
- [x] 1.2.3 Адаптивная дискретизация рёбер: начать с равномерной сетки, затем рекурсивно subdividing по chord deviation
- [x] 1.2.4 Для PCURVE: если `CoEdge.pcurve` есть → использовать его для UV, иначе → `surface.project_point()` — заложена инфраструктура, полная интеграция в Фазе 2.4
- [x] 1.2.5 Интегрировать кэш в `triangulate_face`, `triangulate_shell`, `triangulate_solid` — базовая интеграция через `triangulate_face_cached`
- [ ] 1.2.6 Все граничные точки граней брать ТОЛЬКО из кэша — никакого независимого сэмплирования — требует Phase 2.2 (UV CDT)
- [x] 1.2.7 Unit-тесты: два смежных цилиндра, конус + плоскость, сфера с крышкой — 3 теста в edge_cache.rs
- [ ] 1.2.8 Интеграционный тест: триангуляция Zentralstaender.stp — проверить отсутствие висячих рёбер

**Критерий приёмки:** Для любого `Solid` после триангуляции каждое внутреннее ребро (shared by 2 faces) имеет идентичные 3D-точки с обеих сторон.
